## 当前进展

项目已经从单页技能树升级为本地优先的“产研流程角色工作台”。当前 MVP 支持：

- 按目标输入产研问题，并基于本地规则生成结构化协作路径。

- 展示 skill 更新状态，并用 localStorage 保存已读、解锁、当前入口、当前阶段、当前角色和已生成方案。
- 导出生成方案为 Markdown 和 JSON。

## 下一阶段：v0.2 本地 Skill 图谱

### 阶段目标

把当前写死在数据里的 skill tree 改为从当前设备的本地 skill 目录读取真实 `SKILL.md` 元数据，并用确定性规则按角色、流程、能力深度系统化归类和可视化。v0.2 不依赖 AI、不依赖联网，先把本地 skill 资产变成可探索、可学习的图谱技能树。

### 为什么现在做

当前 MVP 已证明“按目标、按流程、按角色”的信息架构成立，但 skill 数据仍是演示数据。项目真正的价值应来自用户本地已有 skill 资产：让用户看到自己有哪些 skill、它们适合哪个角色和流程阶段、学习路径如何展开。先不用 AI，可以降低不确定性，也让图谱结果可解释、可测试、可跨设备复现。

### 本轮必须完成

- 本地 skill catalog：后端只读扫描本机 skill roots，解析 `SKILL.md` frontmatter、名称、描述、路径和正文摘要。
- 确定性 Skill 分类：基于 frontmatter、正文关键词、目录路径和规则词典，把 skill 映射到角色、流程阶段、能力层级、标签和推荐使用场景。
- 可视化技能树：按角色展示真实本地 skill，并支持按流程阶段切换视角。
- 学习引导：为每个角色/阶段给出从入门到深入的 skill 熟悉路径。
- 按目标规则推荐：用户输入目标后，先用关键词匹配和阶段/角色规则推荐本地 skill 组合，不调用 AI。
- 后端服务层：新增最小 Node API，只负责跨设备本地 skill 发现、读取、分类和状态保存。
- QA 收口：补端到端 smoke test 和真实浏览器视觉 QA，确认本地 skill 能被读取、分类、展示和用于目标推荐。

### 本轮暂缓

- 账号体系、权限、多用户协作。
- 云端部署和域名。
- AI 规划调用路径。
- GitHub 候选检索。
- 可视化数据分析。
- 完整模板编辑器或 skill 市场。
- 复杂长任务队列和流式生成体验。
- 自动安装 GitHub skill。

### 成功标准

- 系统能读取本机 skill roots，并展示真实 `SKILL.md` 列表。
- 每个 skill 能看到名称、描述、来源路径、角色归类、流程阶段归类和推荐深度。
- 按角色视图展示真实本地 skill tree，而不是演示写死数据。
- 按流程视图能显示当前阶段建议熟悉和调用的本地 skill。
- 用户输入目标后，系统能基于本地 skill catalog 和确定性规则推荐相关 skill、角色和流程阶段。
- 推荐结果能展示匹配原因，例如命中的关键词、角色规则或阶段规则。
- 图谱生成不需要 AI key，不需要联网。
- 后端 health check、skill catalog、skill 分类和目标推荐 API 有自动化测试或 smoke test。
- `pnpm test:run`、`pnpm build` 通过。
- 浏览器 QA 文档确认 390x844、768x1024、1440x900 三个视口核心闭环可用。

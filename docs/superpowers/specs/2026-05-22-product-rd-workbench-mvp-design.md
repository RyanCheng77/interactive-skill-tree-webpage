# 产研流程角色工作台 MVP 设计规格

日期：2026-05-22
状态：待评审
项目：Interactive Skill Tree Webpage

## 背景

当前项目是一个 React/Vite 前端原型，核心形态是“技能熔炉”：围绕产品、设计、前端、后端、测试、运维等角色展示互动技能树。现有页面已经具备角色切换、技能节点、依赖解锁、详情面板、安装命令和工具链接，但整体仍偏“工具学习导航”，缺少面向真实产研流程的行动入口。

本次 MVP 将其升级为“产研流程角色工作台”：用户可以从目标、流程、角色三个角度理解和推进产研工作，并把技能树用于连接具体 skill/tool、角色任务和流程阶段。

## MVP 目标

MVP 要回答一个核心问题：

> 当用户要推进一个产研目标时，系统如何给出一条可理解、可执行、可导出的角色协作路径？

第一版聚焦个人使用和本地演示，不做账号、后端、多用户协作、真实 AI 服务调用。推荐结果先用本地规则和预设模板生成，后续可替换为模型能力。

## 目标用户

MVP 面向综合型门户使用场景，覆盖三类用户：

- 产研新人或转岗者：理解不同角色、流程阶段和工具能力。
- 项目负责人或团队 Lead：快速组织一条 MVP 推进路径，发现角色和 skill 缺口。
- AI/Agent 工作流使用者：理解某个产研目标应调用哪些角色、skill 和工具。

## 信息架构

应用采用左侧工作台导航，主导航只有三个入口：

1. 按目标
2. 按流程
3. 按角色

三个入口职责固定：

- 按目标 = 生成。用于 AI 式目标输入、推荐任务模板、结构化生成结果。
- 按流程 = 查看阶段。用于按产研阶段查看角色任务、阶段产物和建议 skill/tool。
- 按角色 = 查看角色。用于按角色查看当前阶段任务、建议 skill/tool、完整技能树和 skill 更新状态。

## 视觉与动效约束

MVP 必须严格延续当前代码里的“技能熔炉”视觉语言，不切换成另一套浅色 SaaS 工作台风格。

必须保留：

- 暗色背景体系：页面、面板、卡片继续使用现有深色层级，例如 `#070710`、`#09090f`、`#0d0d16`、`#12121e`、`#1a1a28`。
- 角色色体系：产品、设计、前端、后端、测试、运维等角色继续使用现有 `color`、`glowColor`、`accentColor`。
- 字体气质：标题继续沿用 `Cinzel`，正文/说明继续沿用 `Crimson Pro`，状态、层级、命令继续沿用 `JetBrains Mono`。
- 技能树动效：选中节点的 glow pulse、可解锁节点的脉冲环、连接线激活态、节点层级徽标等现有动效必须保留或等价迁移。
- 圆角与密度：保持现有 `rounded-lg` 为主的紧凑卡片，不使用大面积浅色留白和过大的圆角作为主风格。
- 按钮和状态：沿用现有的 role color 高亮、暗色 hover、`transition-all duration-150/200`、`active:scale-95` 等动效节奏。

允许调整：

- 为适配左侧工作台和移动端，可以重新组织布局。
- 可以新增“按目标/按流程/按角色”的工作台结构。
- 可以为结果页和推荐任务新增组件，但组件必须使用现有视觉 token 和动效语法。

不允许：

- 将整体 UI 改成白底浅色管理后台。
- 使用与当前代码无关的大面积 slate/gray SaaS 样式作为最终实现。
- 删除技能树的游戏化状态表达。

## 按目标视图

按目标是 MVP 的默认入口，采用常见 AI 入口布局：

- 页面中央提出主问题：“我们该推进什么？”
- 下方是大输入框，支持用户用一句话描述产研目标。
- 输入框底部只保留与产品相关的控制项：
  - 自动规划
  - 输出内容选择，例如“协作路径 + 任务清单”
  - 提交按钮
- 不展示通用 AI 产品里的无关控件，例如本地模式、深度、语音。
- 输入框下方展示推荐任务模板。

推荐任务模板支持“换一批”：

- MVP 用本地预设模板池实现。
- 默认展示 3 个模板。
- 点击“换一批”轮换下一组模板。
- 点击模板后，将模板内容填入输入框或直接触发生成。

首批目标模板包括：

- 从 0 到 1 做 MVP
- 组织一次需求评审
- 推进设计到上线
- 开发联调与上线验收
- 复盘并沉淀项目 skill

## 生成结果页

从按目标生成后，进入结构化结果页，而不是输出一段聊天文本。

结果页是一个“生成方案”主画布：

- 顶部有面包屑：按目标 > 生成结果 > 当前方案。
- 顶部展示目标摘要、重新生成、导出文档。
- 主体采用左右结构：
  - 左侧是阶段目录。
  - 右侧是当前阶段详情。

当前阶段详情包含：

- 阶段说明
- 角色任务
- 对应 skill/tool
- 阶段产物
- 下一步阶段
- 进入按流程查看的入口

结果页底部提供明确下一步：

- 确认方案后进入实施计划。
- 后续通过 `writing-plans` 将结构化结果转换成可执行实施计划。

导出能力：

- Markdown 为主，用于进入项目文档。
- JSON 为辅，用于后续 agent/skill 继续执行。

## 按流程视图

按流程视图参考按角色页的主从结构，避免横向卡片过散。

布局：

- 左侧为流程阶段选择栏。
- 右侧为当前阶段工作台。
- 右上角主按钮为“反馈”，MVP 只保留入口，不实现反馈面板。

阶段包括：

1. 机会发现
2. 需求定义
3. 设计验证
4. 开发实现
5. 测试验收
6. 发布运营
7. 复盘沉淀

当前阶段工作台包含：

- 阶段定位和说明
- 参与角色与任务
- 建议 skill/tool
- 阶段产物
- 输入、输出与下一步
- 导出阶段卡入口

## 按角色视图

按角色视图同样采用主从结构：

- 左侧为角色选择栏。
- 右侧为当前角色工作台。
- 右上角主按钮为“反馈”，MVP 只保留入口，不实现反馈面板。

角色包括：

- 项目负责人
- 产品经理
- 设计师
- 前端工程师
- 后端工程师
- 测试工程师
- 运维工程师

当前角色工作台包含：

- 当前角色说明
- 当前阶段
- 本阶段核心任务
- 建议 skill/tool
- 关键产物
- 完整技能树

技能树必须默认完全展开，不藏在“展开完整技能树”按钮后。可以保留树形/列表切换，但默认展示完整树形能力路径。

## Skill 更新状态

具体 skill 有更新时，技能树必须展示更新状态。

更新状态分三层展示：

- 角色层：角色选择栏显示该角色有几个 skill 更新。
- 节点层：技能树节点显示“新增”“更新”“建议复习”等状态。
- 详情层：点击节点后展示版本变化、更新时间和更新摘要。

MVP 数据字段建议：

- `version`
- `updatedAt`
- `changeSummary`
- `updateType`
- `seen`

更新状态支持标为已读。MVP 可用 localStorage 保存已读状态，不需要后端。

## 本地状态与持久化

MVP 使用 localStorage，不接后端。

需要保存：

- 最近目标输入
- 生成结果
- 当前选中流程阶段
- 当前选中角色
- 推荐任务“换一批”索引
- skill 更新已读状态
- 技能树解锁或浏览状态

## 暂缓到下一期

以下能力不进入本期 MVP：

- 反馈面板的具体交互和数据闭环
- 生成记录列表
- 账号体系
- 后端存储
- 多人协作
- 真实 AI 模型接入
- 权限管理

本期仅保留“反馈”入口语义，以及结果页面包屑中的返回层级，不实现完整记录管理。

## 数据结构草案

```ts
type EntryMode = "goal" | "process" | "role";

interface GoalTemplate {
  id: string;
  title: string;
  description: string;
  prompt: string;
  category: string;
}

interface GeneratedPlan {
  id: string;
  title: string;
  goal: string;
  createdAt: string;
  stages: PlanStage[];
  recommendedSkills: string[];
  deliverables: string[];
}

interface PlanStage {
  id: string;
  name: string;
  summary: string;
  roles: RoleTask[];
  skills: string[];
  deliverables: string[];
  nextStageId?: string;
}

interface RoleTask {
  roleId: string;
  task: string;
  output: string;
  skills: string[];
}

interface Skill {
  id: string;
  name: string;
  roleId: string;
  tier: number;
  prereqs: string[];
  version: string;
  updatedAt?: string;
  changeSummary?: string;
  updateType?: "new" | "updated" | "review";
  seen?: boolean;
}
```

## 验收标准

MVP 完成后应满足：

- 左侧主导航只有按目标、按流程、按角色。
- 按目标默认展示 AI 式输入入口和推荐任务模板。
- 推荐任务支持换一批。
- 按目标生成后进入结构化结果页，不是聊天文本。
- 结果页有面包屑，能返回上一层语义。
- 按流程采用左侧阶段选择栏 + 右侧阶段工作台。
- 按角色采用左侧角色选择栏 + 右侧角色工作台。
- 按角色视图中技能树默认完全展开。
- skill 更新能在角色层、节点层、详情层展示。
- 本地状态用 localStorage 保存。
- 可导出 Markdown 和 JSON。
- 反馈面板和生成记录列表不进入本期。
